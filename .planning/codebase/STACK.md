## Configuration

**Environment:**
- Single source of truth: `src/lib/env.ts` (Zod schema-validated)
- Required variables:
  - `ENCRYPTION_MASTER_KEY` – 64-char hex string for AES-256-GCM encryption of sensitive data
  - `DATABASE_PATH` – Optional; defaults to `~/Library/Application Support/Itsyconnect/itsyconnect.db` on macOS
  - `PORT` – Optional; defaults to 3000
  - AI provider keys (optional, runtime: `ANTHROPIC_API_KEY`, `OPENAI_API_KEY`, `GOOGLE_GENERATIVE_AI_API_KEY`, `XAI_API_KEY`, `MISTRAL_API_KEY`, `DEEPSEEK_API_KEY`, `LOCAL_OPENAI_API_KEY`)

**Build:**
- `next.config.ts` – Next.js configuration (CSP headers, image allowlist for mzstatic.com, standalone output)
- `tsconfig.json` – TypeScript compiler options (ES2017 target, path alias `@/*`)
- `tailwindcss` v4 with `@tailwindcss/postcss` – CSS framework
- `postcss.config.mjs` – PostCSS pipeline with Tailwind v4
- `eslint.config.mjs` – ESLint 9 flat config with Next.js and TypeScript rules
- `vitest.config.ts` – Test runner config (node environment, 100% coverage threshold for lib/**, v8 provider)
- `drizzle.config.ts` – ORM config pointing to SQLite schema and migrations folder

**Versioning:**
- Single source: `src/lib/version.ts` exports `APP_VERSION` ("1.7.0") and `BUILD_NUMBER` ("170")
- `package.json` version must match `APP_VERSION`
- `forge.config.ts` imports `BUILD_NUMBER` for `CFBundleVersion` in Electron bundle

## Dependency management

**Lockfile strategy:**
- npm lockfile tracked in git (`package-lock.json`)
- All dependencies pinned to exact versions via `package-lock.json`

## Platform requirements

**Development:**
- macOS with Xcode toolchain (for native sqlite3 rebuild via `npm rebuild better-sqlite3`)
- Node.js 20+
- npm 10+

**Production (Electron):**
- macOS 10.13+ (Intel or Apple Silicon via universal binary)
- App Store distribution: Code signing with Apple certificates, MAS provisioning profiles, entitlements (`entitlements.mas.plist`, `entitlements.mas.child.plist`)
- Direct macOS distribution: Notarization via Apple ID (env vars: `APPLE_ID`, `APPLE_ID_PASSWORD`, `APPLE_TEAM_ID`)
- Multi-arch universal binary build via `@electron/universal`

**Deployment:**
- Standalone Next.js server via `next build` + `node .next/standalone/server.js`
- Embedded in Electron.app bundle under `Contents/Resources/app/.next/standalone/`
- Native modules (sqlite, shared libs) stored in `.next/standalone/node_modules` for Mach-O detection during universal build
